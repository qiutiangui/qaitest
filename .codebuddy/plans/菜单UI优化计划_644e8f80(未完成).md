## 产品概述

优化 qaitest智测平台 的菜单结构，使AI功能更易识别，并整合相关功能模块。

## 核心功能

### 菜单命名优化

- "需求分析" 改为 "AI需求分析"
- "用例生成" 改为 "AI用例生成"

### 菜单结构整合

**优化前（8个扁平菜单）：**

- 项目管理、需求分析、功能点管理、用例生成、用例管理、测试计划、测试报告、版本管理

**优化后（6个菜单，含2个子菜单组）：**

- 项目管理
- AI需求分析（子菜单：需求分析、功能点管理）
- AI用例生成（子菜单：用例生成、用例管理）
- 测试计划
- 测试报告
- 版本管理

### 视觉效果

- 保持统一风格，不加特殊视觉标识
- 子菜单支持展开/收起交互
- 当前页面高亮显示

## 技术栈

- Vue 3 + TypeScript
- Lucide Vue Next（图标库）
- 纯CSS样式（复用现有设计系统）

## 实现方案

### 菜单结构调整

将扁平菜单改造为支持嵌套子菜单的结构，通过 `children` 字段定义子菜单项。

### 关键数据结构

```typescript
interface MenuItem {
  path?: string           // 路由路径（有子菜单时可为空）
  name: string            // 菜单名称
  icon: Component         // 图标组件
  children?: MenuItem[]   // 子菜单（可选）
}
```

### 实现要点

1. **菜单配置**：修改 `menuItems` 数组，添加 `children` 字段支持子菜单
2. **展开状态**：使用 `expandedMenus` Set 存储已展开的菜单路径
3. **路由匹配**：子菜单项匹配时，父菜单自动高亮并展开
4. **动画效果**：子菜单展开/收起使用 CSS transition

### 目录结构

```
frontend/src/
├── components/
│   └── Layout.vue        # [MODIFY] 菜单配置和渲染逻辑
└── router/
    └── index.ts          # [保持] 路由路径不变
```

## 设计风格

保持现有简约现代风格，沿用白色背景、细边框分隔、hover高亮等设计语言。

## 菜单层级交互

- 一级菜单：点击有子菜单的项时展开/收起子菜单
- 子菜单项：点击跳转对应页面
- 展开图标：有子菜单的项右侧显示展开/收起箭头
- 默认行为：进入子菜单页面时，父菜单自动展开
