## 产品概述

优化AI用例生成流程的实时日志输出和完成区域交互，实现简洁、清晰的界面体验。

## 核心功能

### 1. 实时日志区域简化

- **功能点列表**：只显示"正在加载 N 个功能点..."，不打印具体内容
- **第一版测试用例**：不打印，不落库（仅在内存中处理）
- **评审结论**：只显示"评审完成"，并将评审结论保存到数据库
- **最终测试用例**：只显示"已保存 N 个测试用例"，不打印具体内容

### 2. 评审结论落库

- 将评审结论保存到 TestCaseGenerationTask 表的 result 字段（JSON格式）
- 包含：approved, summary, total_testcases, coverage_rate, issues, suggestions

### 3. 完成区域增强

- **查看功能点**：跳转到功能点列表页面（带项目ID过滤）
- **查看测试用例**：跳转到测试用例列表页面（带ID过滤）
- **导出功能点**：导出功能点Excel（已存在，优化触发方式）
- **导出用例**：导出测试用例Excel（已存在，优化触发方式）

### 4. 完成后数据传递

- 后端在WebSocket完成消息中返回 function_ids（功能点ID列表）
- 前端根据这些ID实现精准跳转和导出

```typescript
// 跳转到功能点列表
const goToFunctionPoints = () => {
  const functionIds = statsInfo.value.functionIds
  if (functionIds && functionIds.length > 0) {
    router.push({
      path: '/requirements',
      query: { ids: functionIds.join(',') }
    })
  } else {
    router.push('/requirements')
  }
}

// 跳转到测试用例列表
const goToTestCases = () => {
  const testcaseIds = statsInfo.value.savedIds
  if (testcaseIds && testcaseIds.length > 0) {
    router.push({
      path: '/testcases',
      query: { ids: testcaseIds.join(',') }
    })
  } else {
    router.push('/testcases')
  }
}

// 格式化时长
const formatDuration = (seconds: number) => {
  if (!seconds) return '0秒'
  if (seconds < 60) return `${Math.round(seconds)}秒`
  const minutes = Math.floor(seconds / 60)
  const secs = Math.round(seconds % 60)
  return secs > 0 ? `${minutes}分${secs}秒` : `${minutes}分钟`
}
```
