## 用户需求

将测试用例的创建者字段从固定的"AI"改为实际生成测试用例的大模型名称（如"deepseek"），动态获取实际使用的模型名称，不写死。

## 核心功能

- 在测试用例生成过程中，自动记录实际使用的大模型名称
- 将模型名称作为创建者保存到测试用例数据中
- 支持不同模型配置时正确显示对应模型名称

## 涉及场景

- AI智能体生成测试用例（Runtime模式和简化模式）
- 手动创建测试用例时保持原有逻辑

## 技术方案

### 实现思路

通过消息传递机制，将模型名称从生成入口函数传递到数据库保存Agent，确保创建者字段使用实际的大模型名称。

### 修改链路

```
API入口 → run_testcase_generation → 各Agent消息传递 → TestCaseInDatabaseAgent → 数据库保存
```

### 关键修改点

1. **消息类型扩展** (`messages.py`)

- `TestCaseReviewMessage` 添加 `creator` 字段
- `TestCaseFinalizeMessage` 添加 `creator` 字段

2. **Agent消息传递** (`testcase_agents.py`)

- `TestCaseReviewAgent` 发布消息时传递 `creator`
- `TestCaseFinalizeAgent` 发布消息时传递 `creator`
- `TestCaseInDatabaseAgent` 使用消息中的 `creator` 保存

3. **Prompt模板修改** (`testcase_agents.py`)

- `TestCaseFinalizeAgent` 的 prompt 中 `"creator":"AI"` 改为动态填充

4. **简化流程同步修改** (`testcase_agents.py`)

- `run_testcase_generation` 函数中保存用例时使用 `gen_model_name`

### 默认值处理

- 当没有模型配置时，使用系统默认模型名称 "deepseek"
- 手动创建测试用例时，creator 由前端传入或保持默认值
