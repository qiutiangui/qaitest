## 产品概述

优化功能点和测试用例管理页面的编辑交互体验，统一弹窗风格，并为测试用例创建独立详情页。

## 核心功能

### 功能点页面优化

- 列表页编辑弹窗：改用 Element Plus 的 el-dialog + el-form 组件，提升视觉效果
- 详情页：保持现有独立页面不变，已支持页面内编辑模式

### 测试用例页面改造

- 列表页：新增编辑按钮和编辑弹窗（使用 Element Plus 组件）
- 详情页：创建独立详情页，参照功能点详情页实现，支持页面内编辑模式
- 列表查看按钮：改为跳转到独立详情页

## 交互逻辑

### 功能点

- 列表"编辑"按钮 → 打开编辑弹窗
- 列表"查看"按钮 → 跳转到独立详情页 → 点击"编辑"按钮 → 页面内切换为编辑模式

### 测试用例

- 列表"编辑"按钮 → 打开编辑弹窗
- 列表"查看"按钮 → 跳转到独立详情页 → 点击"编辑"按钮 → 页面内切换为编辑模式

## 技术栈

- Vue 3 + TypeScript
- Element Plus 组件库（el-dialog, el-form, el-descriptions）
- Tailwind CSS
- 复用现有组件：DetailHeader、DetailCard、EditableField

## 设计原则

### 美观性

1. 使用 Element Plus 组件替代原生HTML元素
2. 统一的间距、颜色、圆角样式
3. 清晰的表单布局和标签
4. 友好的交互反馈

### 风格统一

1. **弹窗风格统一**：

- 参考 `ProjectList.vue` (362-386行) 的实现
- 使用 `el-dialog` + `el-form` + `el-form-item`
- 统一宽度：功能点 600px，测试用例 600px
- 统一按钮样式：取消（btn-secondary）+ 确定（btn-primary）

2. **详情页风格统一**：

- 参考 `RequirementDetail.vue` 的实现
- 复用 DetailHeader、DetailCard、EditableField 组件
- 统一的页面布局和编辑交互

3. **列表页风格统一**：

- 操作按钮位置和样式保持一致
- 图标和文字搭配
- hover 效果统一
