## 产品概述

优化 qaitest智测平台的菜单结构，使AI功能更易识别，整合相关功能模块，并增加需求维度的管理能力。

## 核心功能

### 菜单命名优化

- "需求分析" 改为 "AI需求分析"
- "用例生成" 改为 "AI用例生成"

### 菜单结构整合

**优化前（8个扁平菜单）：**
项目管理、需求分析、功能点管理、用例生成、用例管理、测试计划、测试报告、版本管理

**优化后（6个菜单，含2个子菜单组）：**

- 项目管理
- 需求管理（子菜单：AI需求分析、需求列表、功能点列表）
- 用例生成（子菜单：AI用例生成、用例列表）
- 测试计划
- 测试报告
- 版本管理

### 需求维度增强

- 新增需求列表页面，按 `requirement_name` 分组展示需求
- 功能点列表增加"所属需求"列筛选
- 前端类型定义补充 `requirement_name` 字段

### 视觉效果

- 保持统一风格，不加特殊视觉标识
- 子菜单支持展开/收起交互
- 当前页面高亮显示，父菜单自动展开

## 技术栈

- 前端：Vue 3 + TypeScript + Lucide Vue Next
- 后端：FastAPI + Tortoise ORM
- 样式：Tailwind CSS

## 实现方案

### 1. 菜单结构调整

将扁平菜单改造为支持嵌套子菜单的结构：

```typescript
interface MenuItem {
  path?: string
  name: string
  icon: Component
  children?: MenuItem[]
}
```

### 2. 数据结构发现

后端已有 `requirement_name` 字段（见 `backend/app/schemas/requirement.py` 第11行），但前端类型定义缺失，需要补充。

### 3. 后端API扩展

新增 `GET /api/requirements/groups` 接口，按 `requirement_name` 分组返回需求列表。

## 实现要点

1. **子菜单渲染**：使用递归组件或 v-for 嵌套实现子菜单
2. **展开状态**：使用 `expandedMenus` Set 管理展开状态
3. **路由匹配**：子菜单项匹配时，父菜单自动高亮并展开
4. **需求分组**：后端按 `requirement_name` 聚合，返回分组列表

## 设计风格

保持现有简约现代风格，沿用白色背景、细边框分隔、hover高亮等设计语言。

## 菜单层级交互

- 一级菜单：点击有子菜单的项时展开/收起子菜单
- 子菜单项：点击跳转对应页面
- 展开图标：有子菜单的项右侧显示 ChevronDown/ChevronRight 箭头
- 默认行为：进入子菜单页面时，父菜单自动展开
