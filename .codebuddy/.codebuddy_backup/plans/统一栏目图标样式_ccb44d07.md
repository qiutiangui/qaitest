## 用户需求

功能点详情页、功能点编辑页、测试用例详情页、测试用例编辑页中的栏目标题前的图标需要统一处理：

- 图标不一定要完全相同
- 不能是空的
- 不能是纯背景色图标（当前某些栏目未传 icon 参数，导致只显示渐变背景块）

## 问题分析

DetailCard 组件在没有传 `icon` 参数时，仍渲染带渐变背景的空 div，导致显示纯背景色的"空白图标"。

## 需要添加图标的栏目

**FunctionPoints.vue：**

- 详情弹窗：属性信息、验收标准、关键词
- 编辑弹窗：属性设置、验收标准、关键词

**TestCases.vue：**

- 详情/编辑/新建弹窗：前置条件、测试数据、其他信息

## 技术方案

为每个 DetailCard 组件添加合适的 lucide-vue-next 图标，确保所有栏目标题前都有实际图标而非空白背景。

## 图标选择方案

| 栏目名称 | 推荐图标 | 说明 |
| --- | --- | --- |
| 属性信息/属性设置 | Settings | 表示配置属性 |
| 验收标准 | CheckCircle | 表示验收确认 |
| 关键词 | Tag | 表示标签关键词 |
| 前置条件 | ListChecks | 表示前置检查项 |
| 测试数据 | Database | 表示数据存储 |
| 其他信息 | Info | 表示附加信息 |


## 实现步骤

1. 在 FunctionPoints.vue 导入新图标：Settings, CheckCircle, Tag
2. 在 TestCases.vue 导入新图标：ListChecks, Database, Info
3. 为各弹窗中的 DetailCard 组件添加 :icon 属性
